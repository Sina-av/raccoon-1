StickyBC

Imposes the boundary condition if exceeds the bounds provided

Description

This nodal boundary condition imposes the condition if exceeds either or Hence the name "sticky": as soon as exceeds the bounds it is fixed at subsequent timesteps.

commentnote

StickyBC should be used with care. It only approximates a Constraint imposed by a wall, for instance, for: the boundary nodes may penetrate the wall in a single timestep before they are then stuck (inside the wall); boundary nodes can never move away from the wall again once they are stuck. However, it offers the advantage over a full-blown Constraint that it does not adversely affect numerical convergence.

Input Parameters

  • boundaryThe list of boundary IDs from the mesh where this boundary condition applies

    C++ Type:std::vector<BoundaryName>

    Options:

    Description:The list of boundary IDs from the mesh where this boundary condition applies

  • variableThe name of the variable that this residual object operates on

    C++ Type:NonlinearVariableName

    Options:

    Description:The name of the variable that this residual object operates on

    Options:

    Description:The name of auxiliary variables to save this BC's diagonal jacobian contributions to. Everything about that variable must match everything about this variable (the type, what blocks it's on, etc.)

    C++ Type:std::vector<VariableName>

    Options:

    Description:The displacements

  • max_value1.79769e+308If the old variable value >= max_value, the variable is fixed at its old value

    Default:1.79769e+308

    C++ Type:double

    Options:

    Description:If the old variable value >= max_value, the variable is fixed at its old value

  • min_value-1.79769e+308If the old variable value <= min_value, the variable is fixed at its old value

    Default:-1.79769e+308

    C++ Type:double

    Options:

    Description:If the old variable value <= min_value, the variable is fixed at its old value

  • save_inThe name of auxiliary variables to save this BC's residual contributions to. Everything about that variable must match everything about this variable (the type, what blocks it's on, etc.)

    C++ Type:std::vector<AuxVariableName>

    Options:

    Description:The name of auxiliary variables to save this BC's residual contributions to. Everything about that variable must match everything about this variable (the type, what blocks it's on, etc.)
