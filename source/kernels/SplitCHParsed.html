SplitCHParsed

Split formulation Cahn-Hilliard Kernel that uses a DerivativeMaterial Free Energy

Implements the weak form

where is the concentration variable the kernel is operating on, is a free energy material property (with accompanying derivative properties), is the chemical potential variable, and is the gradient energy coefficient.

It is used together with SplitCHWRes and CoupledTimeDerivative to set up a system of two first order PDEs using a concentration order parameter and a chemical potential variable.

  • f_nameBase name of the free energy function F defined in a DerivativeParsedMaterial

    C++ Type:MaterialPropertyName

    Options:

    Description:Base name of the free energy function F defined in a DerivativeParsedMaterial

  • kappa_nameThe kappa used with the kernel

    C++ Type:MaterialPropertyName

    Options:

    Description:The kappa used with the kernel

  • variableThe name of the variable that this residual object operates on

    C++ Type:NonlinearVariableName

    Options:

    Description:The name of the variable that this residual object operates on

  • wChemical potential

    C++ Type:std::vector<VariableName>

    Options:

    Description:Chemical potential

    Options:

    C++ Type:std::vector<SubdomainName>

    Options:

    Description:The list of block ids (SubdomainID) that this object will be applied

    Options:

    Description:The displacements

  • control_tagsAdds user-defined labels for accessing object parameters via control logic.

    C++ Type:std::vector<std::string>

    Options:

    Description:Adds user-defined labels for accessing object parameters via control logic.

    Default:True

    C++ Type:bool

    Options:

    C++ Type:std::vector<AuxVariableName>

    Options:

    Description:The name of auxiliary variables to save this Kernel's residual contributions to. Everything about that variable must match everything about this variable (the type, what blocks it's on, etc.)

    Default:0

    Options:

    Description:The seed for the master random number generator

    Description:Whether or not this object should use the displaced mesh for computation. Note that in the case this is true but no displacements are provided in the Mesh block the undisplaced mesh will still be used.
